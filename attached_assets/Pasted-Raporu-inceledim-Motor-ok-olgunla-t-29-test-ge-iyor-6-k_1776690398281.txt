> **TASK #157 — Clear originalHostId on intentional manual host transfer**
>
> Currently transferHost() does NOT clear originalHostId. This means if Host A manually gives host to Player B, then disconnects and reconnects, they silently reclaim host from B. This is wrong for intentional transfers.
>
> In transferHost() function in gameEngine.ts:
> ```typescript
> export function transferHost(room: Room, newHostId: string): void {
>   const oldHost = room.players.find(p => p.isHost);
>   if (!oldHost) return;
>   oldHost.isHost = false;
>   const newHost = room.players.find(p => p.id === newHostId);
>   if (!newHost) return;
>   newHost.isHost = true;
>   room.hostId = newHostId;
>   room.originalHostId = undefined; // ← ADD THIS LINE
>   // Intentional transfer clears the auto-restore chain
> }
> ```
